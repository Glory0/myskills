# 日记总结 Skill

从 Obsidian 日记文件夹中提取指定时间范围的笔记，分析 YAML 字段和日期信息，生成结构化的总结报告。

## 使用场景

- 每周工作总结
- 项目回顾
- 时间追踪分析
- 工作负载统计

## 工作流程

### 步骤 1: 确定时间范围

明确需要总结的日期范围：
- 开始日期（如：2026-03-16）
- 结束日期（如：2026-03-18）
- 或者相对时间（如"本周"、"上周"）

### 步骤 2: 检索日记文件

使用 Glob 工具查找指定日期范围的 `.md` 文件：

```
Pattern: 990-日记/2026-03-*.md  （具体月份）
```

### 步骤 3: 读取并分析笔记

对每个笔记文件提取以下信息：

#### 3.1 YAML Frontmatter 字段

| 字段 | 含义 | 数据类型 | 示例 |
|------|------|----------|------|
| `日期` | 笔记归属日期 | Date | `2026-03-16` |
| `跟进` | 后续工作日期列表 | List[Date] | `["2026-03-20", "2026-03-25"]` |
| `DL` | Deadline 截止日期 | Date | `2026-03-20` |
| `DC` | Done Completed 完成时间 | Date | `2026-03-18` |
| `时间` | 记录创建时间 | Time | `10:17` |
| `类型` | 工作类型 | String | `设计-收资` |
| `阶段` | 项目阶段 | String | `施工图` |
| `对接人` | 协作者 | List[String] | `["余杰", "王贵荣"]` |
| `是否完成` | 完成状态 | String | `跟踪中 👀` |
| `请假申请日期` | 外出申请时间 | Date | `2026-03-15` |

#### 3.2 标题中的日期

检查二级标题、三级标题是否包含日期：
```markdown
## 2026-03-16-工作事项名称
## 2026-03-17 会议记录
```

#### 3.3 正文中的日期引用

- 金山文档等外部链接中的日期
- 会议纪要中的时间
- 截止日期描述

### 步骤 4: 分类汇总

#### 4.1 按日期分组

```
2026-03-16: [工作项1, 工作项2, ...]
2026-03-17: [工作项3, 工作项4, ...]
```

#### 4.2 按项目分组

```
汾酒2030: [工作项1, 工作项3]
汾青一期: [工作项2, 工作项5]
```

#### 4.3 按人员分组

```
余杰: [工作项1, 工作项4]
张卓韬: [工作项2, 工作项3]
```

#### 4.4 时间线视图

生成包含 DL、DC、跟进日期的甘特式时间线：

```
2026-03-15 郝丽华申请出差
2026-03-16 张卓韬出差
2026-03-16 工作项A创建 (DL: 2026-03-20)
2026-03-18 工作项A完成 (DC: 2026-03-18)
2026-03-20 工作项A跟进日期
```

### 步骤 5: 生成总结报告

输出到指定文件夹（如 `100-inbox/`），文件名格式：

```
日记总结-YYYY-MM-DD至YYYY-MM-DD-v{N}.md
```

#### 报告结构

```markdown
---
日期: {生成日期}
笔记属性: 总结
tags: 日记总结
---

# 日记总结（{开始日期} 至 {结束日期}）

## 📅 日期索引

## 📋 每日工作详情

### {日期}（星期X）
- 工作记录表格
- 请假/外出记录
- 待处理问题

## 📊 统计汇总
- 按日期分布
- 按项目分布
- 按工作类型分布
- 按状态分布
- 人员参与统计

## ⏰ 时间节点汇总
- DL 汇总
- DC 汇总
- 跟进日期汇总
- 请假/外出时间线

## ⚠️ 待处理事项
- 高优先级
- 跟踪中
- 待持续关注

```

## 字段详解

### 跟进（Follow-up Dates）

```yaml
跟进:
  - 2026-03-20
  - 2026-03-25
```

**含义**: 需要在这些日期继续跟进此工作的进展
**统计方式**: 列出所有有跟进日期的笔记，按跟进日期排序

### DL（Deadline）

```yaml
DL: 2026-03-20
```

**含义**: 工作的截止日期
**统计方式**: 
- 列出所有设置了 DL 的笔记
- 计算距今日期的天数（超期/剩余）
- 标记即将到期（3天内）的事项

### DC（Done Completed）

```yaml
DC: 2026-03-18
```

**含义**: 工作实际完成的时间
**统计方式**:
- 统计完成率（有 DC 的笔记 / 总笔记）
- 计算平均完成周期（DC - 创建日期）
- 识别提前/延期完成

### 日期字段一致性检查

确保以下日期的一致性：
- 文件名日期
- YAML `日期` 字段
- 二级标题日期前缀
- 请假申请日期（如有）

## 版本管理

建议保留多个版本：

| 版本 | 用途 | 保留策略 |
|------|------|----------|
| v1 | 简洁版 | 保留，快速浏览 |
| v2 | 详细版 | 保留，完整分析 |
| v3+ | 修订版 | 保留，后续修改 |

## 最佳实践

1. **日期格式统一**: 确保所有日期使用 `YYYY-MM-DD` 格式
2. **空值处理**: `跟进: ["-", "-"]` 视为空列表
3. **多版本保留**: 每次生成新版本，保留旧版本供对比
4. **链接完整**: 总结中保留所有相关笔记的 wikilink
5. **统计清晰**: 使用表格展示多维度统计数据

## 常见问题

**Q: 如何处理跨日期的工作项？**
A: 以 YAML `日期` 字段为准，在对应日期下列出。如涉及多个日期，在统计时标注"跨日期"。

**Q: 跟进日期与 DL 相同怎么处理？**
A: 在时间节点视图中合并显示，标记为"截止+跟进"。

**Q: 如何识别已逾期事项？**
A: 比较 DL 与当前日期，DL < 今日且无 DC 的事项标记为逾期。
