# Spec Feynman Writer

批量为已生成的规范条笔记生成 `## 费曼` 学习内容，优化 frontmatter，建立条文双链，并执行完整性校验。

## Quick Triggers

- "生成费曼"
- "写费曼"
- "补充费曼内容"
- "优化规范条笔记"
- "给这些条笔记写费曼"

## Workflow

### Step 1: Enumerate target notes
列出需要处理的规范条文件。通常文件名格式为：`规范简写【年份】X.X.X.md`。

### Step 2: Read source notes (MANDATORY)
一次性读取所有目标条笔记。读取时重点观察：
1. **节主题**：该节整体讲什么？各条之间的逻辑关系是什么？
2. **条文正文**：核心规则、数字、限定条件、例外情况。
3. **条文说明**：背景、依据、解释性内容（大白话解释和类比的素材来源）。
4. **图片引用**：`![[...]]` 如果存在，搜索并读取图片，把图表数据转写成文字描述。
5. **已有 wikilinks**：`[[...]]` 指向的其他条或概念，需要在 `### 条文关联` 中引用。

### Step 3: Craft content per article
对每一条笔记，生成以下内容：

#### 3.1 简述（15–30 字）
- 不是复制第一句，而是**提炼核心规则**。
- 例："储罐基础荷载分为永久荷载和可变荷载两类。"
- 例："容积大于50000m³的储罐基础抗震设防为乙类，其余为丙类。"

#### 3.2 aliases
- 提取关键词，逗号分隔。
- 包含专业术语、数字阈值、规范简称等，便于 Obsidian 搜索和双链发现。

#### 3.3 `## ` 标题
- 条笔记模板中通常有一个空的 `## \n` 或原始长句标题。
- 将其替换为**简述**，作为该条笔记的 H2 标题。

#### 3.4 `## 费曼`（5 个固定小节）
必须包含以下小节，缺一不可：

1. **### 一句话总结**  
   用一句大白话概括条文核心，避免术语堆砌。

2. **### 大白话解释**  
   - 2–4 个要点，拆解逻辑、关键数字、适用场景。
   - 如果条文含表格，用 Markdown 表格重述关键数据（表格前必须空一行）。
   - 如果读取了图片中的图表，用文字描述关键规律。

3. **### 条文关联**  
   - 列出与本条相关的同节或其他节条文。
   - **所有条文号必须使用 Obsidian wikilink alias 语法**：`[[钢储罐地规【2008】3.3.2|3.3.2]]`
   - 说明它们之间的逻辑关系（如"3.3.1 给食材，3.3.2 给菜谱"）。

4. **### 类比**  
   `>` 引用块形式。用日常生活比喻帮助记忆。

5. **### 记忆口诀**  
   `>` 引用块形式。简短、押韵、有节奏感。

### Step 4: Batch apply with script
当条目较多时（≥3 条），建议写一个临时 Python 脚本统一写入，避免手动逐条修改出错。

脚本要点：
1. 用字典存储每条的内容（`简述`、`aliases`、`费曼内容`）。
2. 正则替换 frontmatter 的 `简述:` 和 `aliases:` 行。
3. 精确匹配 `## \n`（空 H2）并填入简述，**不要误匹配 `#### 条文说明` 中的 `## `**。
4. 替换 `## 费曼` 到 `## 同节规范` 之间的占位内容为生成的费曼内容。

参考脚本模板见 `scripts/batch_feynman.py`。

### Step 5: Verify & fix
批量写入后必须校验，常见错误：

| 检查项 | 方法 | 修复方式 |
|:---|:---|:---|
| `#### 条文说明` 是否被误替换 | `grep '^#### '` 查看 | 被替换的改回 `#### 条文说明` |
| 简述是否为空 | 检查 frontmatter | 手动补充 |
| `## ` 标题是否仍为空白 | 检查每个文件第 1 个 H2 | 填入简述 |
| 表格前是否缺少空行 | 搜索 `\|` 并看其前一行 | 插入空行 |
| wikilink 中 pipe 是否未转义（在表格内） | 检查表格单元格中的 `[[...\|...]]` | 补反斜杠 |
| 费曼是否有缺小节 | 检查 5 个小节标题是否齐全 | 补充缺失部分 |

## Obsidian Markdown Rules (MANDATORY)

### Tables
- **表格前必须有一个空行**，否则 Obsidian 不渲染。
- 表头后必须有分隔行 `|---|---|`。
- **表格内的 wikilink alias 必须转义 pipe**：`[[note\|alias]]`。

### Headings
- 费曼 5 个小节标题层级固定为 `###`。
- 不要混用 `## 费曼` 和内部的 `###` 以外的层级。

### Wikilinks
- 所有引用到的规范条必须使用 alias 语法：`[[规范文件名|显示名]]`。
- 禁止纯文本写条文号（如"参见 3.3.2"），应改为 wikilink。

## Script Template

`scripts/batch_feynman.py` 提供了一个可复用的脚本模板，步骤如下：
1. 复制并重命名该脚本。
2. 在脚本中编辑 `updates` 字典，填入每条的内容。
3. 运行脚本。
4. 执行 Step 5 的校验。

## Post-run Checklist

- [ ] 所有条笔记的 frontmatter `简述` 已填充且精简
- [ ] 所有条笔记的 `aliases` 已填充
- [ ] 所有条笔记的 `## ` 标题不为空
- [ ] `#### 条文说明` 未被误替换
- [ ] `## 费曼` 包含完整的 5 个小节
- [ ] 条文关联中的双链格式正确（`[[文件|别名]]`）
- [ ] 表格前有空行，表格内 wikilink pipe 已转义
